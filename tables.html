Khởi tạo Bảng & Ràng Buộc (Constraints)

Trong SQL Server, việc thiết kế bảng không chỉ dừng lại ở tên cột và kiểu dữ liệu, mà còn phải đảm bảo Toàn vẹn dữ liệu thông qua các Ràng buộc (Constraints). Việc khai báo đúng các ràng buộc giúp ngăn chặn những dữ liệu "rác" hoặc không hợp lệ được lưu vào hệ thống.

1. Các loại Ràng Buộc cơ bản

  • PRIMARY KEY (Khóa chính): Đảm bảo dữ liệu trong cột là duy nhất và không được NULL. Dùng để định danh mỗi dòng dữ liệu.
  • FOREIGN KEY (Khóa ngoại): Đảm bảo tính toàn vẹn tham chiếu. Dữ liệu cột này phải tồn tại ở khóa chính của bảng khác.
  • CHECK (Ràng buộc miền giá trị): Cực kỳ quan trọng! Dùng để giới hạn dữ liệu nhập vào phải thoả mãn một điều kiện cụ thể (VD: Tuổi >= 18, Điểm >= 0).
  • UNIQUE (Duy nhất): Dữ liệu không được trùng lặp (nhưng được phép có 1 giá trị NULL).
  • DEFAULT (Mặc định): Gán giá trị mặc định nếu người dùng không nhập.

2. Khai báo Ràng Buộc CHECK

Khi học phần tạo bảng, giảng viên thường rất hay yêu cầu sử dụng CHECK để kiểm tra tính hợp lệ của dữ liệu. Bạn có thể khai báo CHECK ngay lúc tạo bảng (CREATE TABLE) hoặc thêm vào sau (ALTER TABLE).

Ví dụ: Tạo bảng với các ràng buộc

CREATE TABLE SINHVIEN (
    MaSV VARCHAR(10) PRIMARY KEY, -- Khóa chính
    HoTen NVARCHAR(100) NOT NULL,
    GioiTinh BIT DEFAULT 1, -- Mặc định là Nam (1)
    Tuoi INT CHECK (Tuoi >= 18 AND Tuoi <= 60), -- Ràng buộc CHECK tuổi
    DiemThi FLOAT CHECK (DiemThi >= 0 AND DiemThi <= 10) -- Ràng buộc CHECK điểm
)

Cú pháp thêm Ràng buộc CHECK (ALTER TABLE)

Nếu bạn đã tạo bảng rồi và quên mất ràng buộc, bạn có thể bổ sung bằng lệnh sau:

ALTER TABLE DIEMHP 
ADD CONSTRAINT CHK_DiemHP CHECK (DiemHP >= 0 AND DiemHP <= 10);

ALTER TABLE SINHVIEN
ADD CONSTRAINT CHK_GioiTinh CHECK (GioiTinh IN (0, 1));
