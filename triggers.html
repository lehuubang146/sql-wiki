3. Triggers (Bẫy Lỗi Tự Động)

Trigger là một loại Stored Procedure ĐẶC BIỆT. Thay vì bạn phải gọi EXEC như Procedure bình thường, thì Trigger sẽ tự động chạy ngầm mỗi khi có ai đó dám INSERT, UPDATE, hoặc DELETE dữ liệu vào một bảng nào đó.

2 Bảng Ma Thuật (Magic Tables)

Bất cứ khi nào Trigger chạy, SQL Server tự động tạo ra 2 bảng ảo chỉ tồn tại trong chớp mắt (giống hệt cấu trúc của bảng đang bị tác động):

  • Bảng inserted: Chứa dữ liệu MỚI vừa được người ta nhập vào (khi INSERT hoặc UPDATE).
  • Bảng deleted: Chứa dữ liệu CŨ vừa bị người ta xóa đi hoặc dữ liệu trước khi sửa (khi DELETE hoặc UPDATE).

Nhiệm vụ của bạn là SELECT dữ liệu từ 2 bảng này ra để kiểm tra.

3.1. AFTER Trigger và Lệnh Phục Hồi

Dùng AFTER INSERT, UPDATE, DELETE nghĩa là thao tác đã xảy ra rồi Trigger mới nhảy vào kiểm tra. Nếu thấy sai luật, ta dùng lệnh ROLLBACK TRANSACTION để hủy bỏ thao tác đó, phục hồi dữ liệu lại như cũ.

-- Yêu cầu: Ngăn chặn nhập điểm lớn hơn 10 hoặc nhỏ hơn 0
CREATE TRIGGER trg_KiemTraDiemHopLe
ON DIEMHP
AFTER INSERT, UPDATE
AS
BEGIN
    DECLARE @Diem FLOAT
    
    -- Lấy điểm số mới vừa bị nhập vào từ bảng inserted
    SELECT @Diem = DIEMHP FROM inserted
    
    -- Nếu điểm tào lao
    IF (@Diem < 0 OR @Diem > 10)
    BEGIN
        PRINT N'Lỗi: Điểm phải từ 0 đến 10.'
        -- Kích hoạt cỗ máy thời gian, quay ngược lại quá khứ, hủy thao tác nhập điểm
        ROLLBACK TRANSACTION
    END
END
GO

3.2. INSTEAD OF Trigger

Dùng INSTEAD OF nghĩa là "Thay thế". Thao tác của người dùng sẽ bị chặn lại và không bao giờ được thực hiện, thay vào đó SQL Server sẽ chạy đoạn code của bạn.

-- Yêu cầu: Ngăn không cho xóa môn học đã có người đăng ký
CREATE TRIGGER trg_NganXoaHocPhan
ON DMHOCPHAN
INSTEAD OF DELETE
AS
BEGIN
    DECLARE @MaHP VARCHAR(10)
    SELECT @MaHP = MAHP FROM deleted -- Lấy môn học đang bị dọa xóa
    
    IF EXISTS (SELECT * FROM DIEMHP WHERE MAHP = @MaHP)
    BEGIN
        PRINT N'Lỗi: Đã có sinh viên đăng ký môn này, bạn không thể xóa.'
        -- Không cần ROLLBACK vì INSTEAD OF chặn đứng lệnh xóa ngay từ đầu rồi
    END
    ELSE
    BEGIN
        -- Trót chặn lệnh xóa rồi nên giờ môn học nào an toàn thì mình phải tự code lệnh xóa
        DELETE FROM DMHOCPHAN WHERE MAHP = @MaHP
    END
END
GO

3.3. Hàm UPDATE(Tên_Cột)

Hàm này trả về TRUE nếu có ai đó đang sửa vào đúng cái cột đó.

-- Yêu cầu: Không cho sửa mã sinh viên (khóa chính)
CREATE TRIGGER trg_BaoVeKhoaChinh
ON SINHVIEN
AFTER UPDATE
AS
BEGIN
    -- Nếu người ta đang sửa cột MASV
    IF UPDATE(MASV)
    BEGIN
        PRINT N'Lỗi: Không được phép đổi mã sinh viên.'
        ROLLBACK TRANSACTION
    END
END
GO

Hoàn thành bài học để tăng tiến độ khóa học SQL của bạn!
