Lập Trình SQL: Hàm (Functions) & Con Trỏ (Cursors)

Bên cạnh Stored Procedures, Hàm (Functions) và Con trỏ (Cursors) là những cấu trúc lập trình mạnh mẽ giúp xử lý logic phức tạp, thao tác trên từng dòng dữ liệu và tái sử dụng mã hiệu quả.

1. Hàm (Function)

Hàm là một đối tượng trong hệ quản trị CSDL, tương tự như thủ tục. Điểm khác biệt lớn nhất là hàm bắt buộc phải trả về một giá trị (một số, chuỗi, hoặc một bảng) và có thể sử dụng trực tiếp trong câu lệnh SELECT.

  • Scalar-valued Function: Giá trị trả về là kiểu dữ liệu cơ sở (int, varchar, float, datetime...).
  • Table-valued Function: Giá trị trả về là một Bảng (Table) có được từ một câu truy vấn.

Cú pháp tạo Hàm (Scalar)

CREATE FUNCTION tên_hàm (@tham_số kiểu_dữ_liệu) 
RETURNS kiểu_trả_về
AS 
BEGIN
    -- Các câu lệnh xử lý
    RETURN giá_trị
END

Ví dụ: Hàm tính điểm trung bình chung

CREATE FUNCTION dbo.DIEM_SV(@MaSV varchar(11), @HocKy char(2)) 
RETURNS numeric(4,1)
AS
BEGIN
    DECLARE @DiemTBC numeric(4,1)
    
    IF NOT EXISTS (SELECT * FROM DIEMHP WHERE MaSV=@MaSV)
        RETURN 0 
        
    SET @DiemTBC = (
        SELECT SUM(DiemHP * Sodvht) / SUM(Sodvht) 
        FROM DMHOCPHAN
        INNER JOIN DIEMHP ON DMHOCPHAN.MaHP = DIEMHP.MaHP
        WHERE HocKy = @HocKy AND MaSV = @MaSV
    ) 
    RETURN @DiemTBC
END

Cách gọi hàm: SELECT MaSV, HoTen, dbo.DIEM_SV(MaSV, 1) FROM SINHVIEN

2. Con Trỏ (Cursor)

Con trỏ là một cấu trúc dữ liệu ánh xạ đến một danh sách kết quả (từ câu lệnh SELECT), cho phép bạn duyệt tuần tự từng dòng (row-by-row) để xử lý logic phức tạp mà các câu lệnh SET-based (như UPDATE, DELETE toàn bảng) khó thực hiện được.

Vòng đời của Con Trỏ

  1. DECLARE: Định nghĩa con trỏ.
  2. OPEN: Mở con trỏ, thực thi câu lệnh SELECT.
  3. FETCH NEXT: Đọc từng dòng dữ liệu vào các biến cục bộ.
  4. Xử lý: Dùng vòng lặp WHILE @@FETCH_STATUS = 0 để lặp.
  5. CLOSE: Đóng con trỏ.
  6. DEALLOCATE: Giải phóng vùng nhớ.

Ví dụ: Dùng Con trỏ để sinh Số Báo Danh tự động

Thuật toán: Quét qua tất cả sinh viên, với mỗi sinh viên, tăng biến đếm lên 1 và ghép với chuỗi '0000' để tạo thành SBD 4 chữ số (ví dụ: 0001, 0002...).

CREATE PROCEDURE DanhSBD 
@MaLop varchar(5) 
AS
BEGIN
    -- Bước 1: Khai báo
    DECLARE cur_SBD CURSOR FOR 
    SELECT MaSV FROM SINHVIEN WHERE MaLop = @MaLop ORDER BY HoTen
    
    -- Bước 2: Mở con trỏ
    OPEN cur_SBD
    
    DECLARE @MaSV varchar(5), @i int 
    SET @i = 1
    
    -- Bước 3: Đọc dòng đầu tiên
    FETCH NEXT FROM cur_SBD INTO @MaSV 
    
    -- Bước 4: Vòng lặp duyệt
    WHILE @@FETCH_STATUS = 0 
    BEGIN
        -- Xử lý: Cập nhật SBD cho sinh viên hiện tại
        UPDATE SINHVIEN SET 
        SBD = RIGHT('0000' + LTRIM(STR(@i)), 4)
        WHERE MaSV = @MaSV
        
        -- Đọc dòng tiếp theo
        FETCH NEXT FROM cur_SBD INTO @MaSV
        SET @i = @i + 1
    END
    
    -- Bước 5 & 6: Đóng và Giải phóng
    CLOSE cur_SBD 
    DEALLOCATE cur_SBD
END
Lưu ý về Hiệu năng: Con trỏ xử lý dữ liệu theo kiểu duyệt từng dòng (RBAR - Row By Agonizing Row) nên sẽ rất chậm so với truy vấn tập hợp (Set-based) thông thường. Chỉ nên dùng Cursor khi không thể giải quyết bằng JOIN, UPDATE ... FROM hoặc các hàm Window Functions (như ROW_NUMBER()).
